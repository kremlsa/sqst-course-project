# Сравнение SAST-инструментов

Проект: `vulnerable-app` | Версия: `1.0-lesson15`

---

## Результаты анализа одного проекта

| Инструмент | Найдено issues | Время | False Positives | Уникальные находки |
|-----------|----------------|-------|-----------------|--------------------|
| SonarQube CE | 12 | 45 сек | 2 | SAST + Code Smells + Coverage |
| Semgrep (custom) | 8 | 8 сек | 1 | Быстро, настраиваемые правила |
| Semgrep (OWASP) | 10 | 12 сек | 3 | OWASP-маппинг из коробки |
| Bandit | 9 | 5 сек | 2 | Python-специфично, легковесный |

---

## Когда что использовать

| Сценарий | Рекомендуемый инструмент |
|----------|------------------------|
| Корпоративный стандарт, метрики, репорты | **SonarQube** |
| Быстрый pre-commit hook | **Semgrep** или **Bandit** |
| Кастомные правила под проект | **Semgrep** |
| Анализ зависимостей на CVE | **OWASP Dependency-Check** |
| Открытый код, GitHub Actions | **CodeQL** (GitHub) или **Semgrep** |

---

## Итог: рекомендуемый стек

```
Pre-commit:  Semgrep (кастомные правила) — 5-10 сек
CI/CD:       SonarQube (глубокий анализ) + OWASP Dep-Check
Scheduled:   Semgrep OWASP ruleset (еженедельно)
```
